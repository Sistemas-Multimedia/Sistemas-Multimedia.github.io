1 Description

Humans are quite eﬃcient recognizing the information stored in images (and frames of a video), even when this information has been degraded or (partially) lost. Quantization [1, 3] is a technique that can remove the part of the visual information that is less relevant for us, and implies lossy coding, which provides compression ratios usually at least one order of magnitude higher than using lossless coding.

In this milestone we will quantize a frame in the RGB color domain. A quantizer (typically denoted by Q) is a encoding system that inputs a sequence of (digital) samples x = {xi} and outputs a sequence of quantization indexes k (see the Fig. 1). The inverse system, called a dequantizer (denoted by Q−1), recovers an approximate version of x that we will denote by x~, whose similarity with x inversely depends on a frame quantization step (QS) Δ. Notice that although Δ is an input parameter to the quantizer, it has not been considered in the ﬁgure to keep it simpler.

We deﬁne the quantization error1

ei = xi −x~i, (1)

that obviously, should be minimized.


Figure 1: Scalar quantization and dequantization of a signal.

Quantizers can be classiﬁes into:

  1. Scalar quantizers: those that produce a quantization index ki by each input sample xi.
  2. Vector quantizers: that process the input by blocks of samples, called vectors, producing a quantization index by vector, and usually, the length of the quantization index y much shorter than the length of the vector. Vector Quantization (VQ) can remove correlation and therefore, are more eﬃcient that Scalar Quantization (SQ). Unfortunately, the computational requirements of VQ are, by far, higher than the needed by SQ. If we also consider that there are other techniques (such as transform coding) that are able to decorrelate the samples with less computation than VQ, we can understand why SQ has been selected, for example, in most video codecs.

Quantizers, also, can be classiﬁed as:

  1. Uniform quantizers: those in which Δ is idependent on the amplitude of the samples.
  2. Non-unifom quantizers, on the contrary, when Δ depends (directly or indirectly) on the amplitude of the samples. An example is a companded quantizer.

Non-uniform quantizers can be:

  1. Static quantizers: if Δ is known a priori.
  2. Adaptive quantizers: when Δ is adapted to minimize the quantization error, depending “on-the-ﬂy” on the characteristics2 of the signal.

All quantizers can be classiﬁed into:

  1. Mid-tread quantizers, if x~i can be 0.
  2. Mid-rised quantizers, if x~i never is 0, even if xi = 0.

Finally, quantizers can be classiﬁed as:

  1. Dead-zone quantizers, that are characterized by a QS of length 2Δ for xi = 0. Deadzone quantizers tends to remove the electronic noise (that usually has an small amplitude compared to the input signal x). Notice that dead-zone quantizers should not be consdered uniform, and that all dead-zone quantizers, by deﬁnition, are mid-tread.
  2. No dead-zone quantizers, when the dead-zone does not exist.

1.1 Quantization in the RGB domain

Supposing that we will use a static uniform dead-zone quantizer, a color RGB image can be quantized chanel by channel, using QSs ΔR, ΔG, and ΔB. A reasonable question that arises here is: given a target bit-rate R for the compressed frame, how the QSs should be chosen to minimize the distortion? At this point we can consider two diﬀerent optimization perspectives. In the ﬁrst one, we consider strictly visual considerations, and obviously, any alternative diﬀerent from

ΔR = ΔG = ΔB (2)

will produce some alteration in the color (also called the “chroma”) of the reconstructed frame.

In the second one, only a pure Rate/Distortion (RD) performance is considered. From a RD perspective, the best combination of QSs is those that optimizes (makes closer to the origin of coordinates) the RD curve. A RD curve expresses the distortion (typically the Root Mean Square Error (RMSE)) as a function of the bit-rate. In the case of encoding a RGB image, we have 3 RD curves, one for each channel. Thus, for example, the RD curve for the red channel measures, for diﬀerent QSs, the distortion of reconstructing only the red channel as a function of the bit-rate generated for the corresponding QS.


Figure 8: Two RD curves with diﬀerent shape.

Normal RD curves are convex (see Fig. 8), which means that if λi is the slope of the curve measured at the i-th point of the curve (starting at the lowest bit-rate), it always hold that

λi > λi+1. (3)

λ quantiﬁes the trade-oﬀ between decreasing the distortion (the slopes are always negative) while the bit-rate increases (the higher the slope, the high the beneﬁt). If we suppose now that the contribution to the quality of each channel are additive, that is

D = DR + DG + DB, (4)

where D denotes distortion, then the optimal QSs must satisfy that [2, 1]

λR = λG = λB. (5)

To see this, lets suppose that we have used, for example, a set of QSs so that λR = 2λG = 2λB, and that we still have room for spending more bits encoding the frame. In this situation, the maximum beneﬁt would be obtained if and only if we decrease ΔR because the sloper is double the slope of the other curves. Therefore, the optimal QSs are obtained when Eq. 5 is true.

Unfortunately, the previous procedure implies the computation of the RD curve for each channel, which is a time-consuming operation. For this reason, and supposing that the statistics of each channel are similar and therefore, each channel is going to generate a RD curve with the same slopes for the same QSs, we can suppose that Eq. 2 satisﬁes Eq. 5.
