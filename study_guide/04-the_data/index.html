1 Description

This milestone shows the structure of the videos used in the MRVC project. Basically, a video is a sequence of (digital) images, that we prefer to name frames. In the context of MRVC, a frame can be described as a matrix of pixels.

We are going to store a video as a sequence of ﬁles with names <preﬁx>000.png, <preﬁx>001.png, ..., where <preﬁx> speciﬁes the video name and “.png” indicates that the frames are coded using the lossless PNG image format [2]. PNG uses DPCM and the DEFLATE codec (Huﬀman coding + LZSS) [1] to remove the spatial and statistical redudancy. Notice that no color transform is used.

Finally, notice that all the frames of a video:

  1. Have the same spatial resolution.
  2. Are represented in the RGB domain.
  3. Each color component (channel) has bit depth of 16 bits, representing an unsigned integer.
  4. The pixel intensities are displaced to the center of the range of values ([0, 65535]) that the components can take (for example, in disk, the component intensity 0 is represented by 32768).
