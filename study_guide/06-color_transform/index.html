1 Description

1.1 Introduction

The main objective of this milestone is to remove the intercomponent redundancy that most of natural images exhibit. In general, redundancy in signals is usually expressed as a correlation dependency between samples. In the case of an image (and of course, of a video), a pixel (a sample) is represented usually with three color components (or channels), each one measuring the energy in a diﬀerent band of the visible spectrum, and one source of redundancy is the possible correlation between those color components.

To estimate the redundancy there are basically two options:

  1. To compute the 0-order (memoryless sources) entropy of the signal: the higher the entropy, the lower the redudancy. In fact, if we suppose that the samples of the signal are uncorrelated, the 0-order entropy is an exact mesarure of the expected bit-rate achieved by an arithmetic encoder. Unfortunately, the 0-order entropy is usally a lower bound for the estimation of the redundancy.
  2. A better way is to use an lossless compressor: the higher the length of the compressed ﬁle compared to the length of the original ﬁle, the lower the redundancy.1 Notice, however, that although this estimation is more accurate than the entropy, in general, it depends on the encoding algorithm (diﬀerent algoritms can provide diﬀerent estimations).

In this milestone we are going to remove the intercomponent redundancy of a video frame X, i.e., the redundancy that may exist between the color components of each pixel of X. The color spaces that we are goint to compare are: (1) RGB, (2) the YCrCb, and (3) the YCoCg.

1.2 RGB/YCrCb

1.2.1 Transforms

To convert a (color) pixel from the RGB into the YCrCb domain, we use the RGB-to-YCrCb (analysis) transform [1]

[ Y Cr Cb ] = [ 0.299 0.587 0.114 0.5 −0.4187 −0.0813 −0.1687 −0.3313 0.5 ] [ R G B ] , (1)

where Y is the luminance (luma) component and CrCb is the crominance (chroma). The main reason for such a mapping is that the HVS (Human Visual System) is much less sensitive to high-frequency components in chroma. Thus, compression systems such as JPEG can downsample the chroma components (usually by 2:1 in each of the horizontal and vertical directions), as well as increase their quantization step sizes with respect to luma, to achieve further compression.

Eq. 1 can be also written as

Y = 0.299R + 0.587G + 0.114B Cr = 0.713(R −Y) + δ Cb = 0.564(B −Y) + δ, (2)

where,

δ = { 128 for 8 bits (unsigned) images, 32768 for 16 bits (unsigned) images, 0.5 for ﬂoating point ([0,1]) images. (3)

is used to avoid negative coeﬃcients. As it can be seen, Cr and Cb are scaled versions of R −Y and B −Y, so Cr and Cb can be interpreted as measures of how much red and blue content in a pixel diﬀers from luma, respectively. Notice also that for a gray pixel, R = G = B = Y, and so Cr = Cb = 0 [1].

The inverse (synthesis) transform is deﬁned by

[ R G B ] = [ 1 1.402 0 1 −0.714 −0.344 1 0 1.772 ] [ Y Cr Cb ] , (4)

or alternatively, by

R = Y + 1.403(Cr − δ) G = Y − 0.714(Cr − δ) − 0.344(Cb − δ) B = Y + 1.773(Cb − δ). (5)

1.2.2 Quantization

After analyzing the frame (representing it in the YCrCb domain), the next natural step is quantization. Supposing that we will use a static uniform dead-zone quantizer with quantization steps ΔY, ΔCr, and ΔCb, for the coeﬃcients Y, Cr, and Cb, repectively, and supposing that the contribution to the reconstruction of X of one of the coeﬃcients is not inﬂuenced by the contribution of the rest of coeﬃcients (for this, both color spaces (RGB and YCrCb) should be orthogonal), the optimal quantization steps ΔY∗, ΔCr∗, and ΔCb∗, can be found using the Constant-Slope Rate-Distortion Quantization Strategy (CS-RS-QS) [3, 2].

As it can be seen in this notebook, a RD (Rate-Distortion) curve is a 2D graph where we represent the distortion generated by the quantization as a function of the bit-rate of the quantization indexes. Thus, the closer the curve to the point (0,0) of the graph, the better the performance of the encoding system in RD terms. Now, if we suppose that each component (Y, Cr, and Cb) is quantized and compressed independently, we can ﬁnd the optimal quantization steps, given a maximum target bit-rate Rmax, selecting them as

λY = λCr = λCb, (6)

where λ(R) is the slope of the RD curve for a given bit-rate R, satisﬁying also that

RY + RCr + RCb ≤ Rmax. (7)

Unfortunately, the RGB-to-YCrCb transform is not orthogonal (for example, in Eq. 2, the value of Cr depends on the value of Y, and therefore, there is a dependency between both basis)2 and therefore neither the RGB and the YCrCb spaces. This diﬁcults the ﬁnding of ΔY∗, ΔCr∗, and ΔCb∗ because the quantization error generated in one of the components inﬂuences the quantization error of the rest of components, and when this happens, we cannot use CS-RS-QS.

Anyway, as you can see in this notebook, the use of the YCrCb color domain can be beneﬁcial, even using a simple quantization strategy such as

ΔY = ΔCr = ΔCb. (8)

As it can be seen, the RD curves can be improved for most bit-rates, and therefore, it can be an interesting tool for removing the intercomponent redundancy from a pure mathematical point of view.

1.3 The RGB/YCoCg transform

Eqs. 1 and 3 were derived by Principal Component Analysis (PCA) on old3 video data. The same procedure has been carried out with newer4 videos, obtaining

[ Y C 1 C2 ] = [ 1 3 1 3 1 3 1 2 0 −1 2 −1 4 1 2 −1 4 ] [ R G B ] ⇔ [ R G B ] = [ 1 1 −2 3 1 0 4 3 1 −1 −2 3 ] [ Y C 1 C2 ] . (9)

Clearly, orthogonality is a desired property in compression systems. For this reason, other similar (but orthogonal) transforms has been proposed. One of the proposed color spaces that is orthogonal to the RGB one is the YCoCg color space. The analysis transform is

[ Y Co Cg ] = [ 1 4 1 2 1 4 1 2 0 −1 2 −1 4 1 2 −1 4 ] [ R G B ] , (10)

and the synthesis transform is

[ R G B ] = [ 1 1 −1 1 0 1 1 −1 −1 ] [ Y Co Cg ] . (11)
