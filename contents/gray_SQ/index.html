1 Objective

This task shows how to implement a simple lossy (gray-scale) image compressor [1] based on Scalar Quantization [4, 5, 6] and PNG [3]. The proposed compressor (the PNG-encoding of the scalar quantization of an gray image) basically removes the number of different gray tones in a gray-scale image (normally, humans hardly distinguish more than 64 different colors). Therefore, the main objective of this task is to analyze the impact in the encoding performance (in terms of a RD (Rate/Distortion) curve [2]) when this kind of visual redundancy (the limited number of recognizeable gray tone levels) is exploited.

2 Uniform scalar quantization of gray-scale pixel intensities

Uniform Scalar Quantization (USQ) transform each input signal value \(\mathbf {s}_i\) in a quantization index \(\mathbf {k}_i\), without considering the rest of samples, and in general, without any other information about \(\mathbf {s}\) [4].

There are different mapping procedures to convert \(\mathbf {s}\) into \(\mathbf {k}\), using USQ (the notetooks gray_SQ_deadzone.ipynb, gray_SQ_midrise.ipynb, gray_SQ_midtread.ipynb) shows the RD performance of different USQs). As you can see, the way in which the visual information is quantized has an influence in the RD performance.

3 Non-Uniform SQ of gray-scale pixel intensities

Uniform quantizers are efficient (minimize the distortion) only if the input samples (gray-scale values of the pixels) are uniformly distributed among the quantization bins [4]. However, when the probability of the samples is not “flat” (or the number of bins is very small compared to the possible number of input values), we can use better quantizers. For example, if we know that most of the samples are small integer values (positive and negative), a quantizer such as gray_SQ_companded.ipynb can be more suitable than an uniform one. In general, when the samples follow a probability distribution that only can be modeled using the PDF (Probability Density Function) of the image, then an efficient USQ is gray_SQ_LloydMax.ipynb.

4 Comparison

As it can be checked in compare_quantizers.ipynb, in general, the better quantizer is the Lloyd-Max quantizer because it adapts the QSSs (Quantization Step Sizes) to the PDF (Probability Density Function) of the image. However, notice also that, since the QSSs depend on the PDF of the image, the PDF or the correspondence between the quantization indexes and the corresponding representation levels (the “pallete”) must be also “transmitted” to the decoder, which decreases the final RD performance.

Notice also the remarkable perormance of the DeadZone Quantizer (DZQ) at low bit-rates. This is one of the reasons why the DZQ is used in several standard image and video codecs (such as JPEG 2000 and H.264). Another reason is that, when the QSSs are powers of two, we can use an bit-plane-based entropy encoder.

Finally, notice that (you must see the notebooks) when the QSSs are powers of 2, the points of the RD curve tend to belong to the convex hull of the RD curve. On the contrary, if the QSSs can be any, the RD curves can have points for bit-rates \(i\) and \(i+1\) where the condition \(\lambda _i\ge \lambda _{i+1}\) (being \(\lambda _i\) the slope of the RD curve at the bit-rate \(i\)) is not satisfied, which can difficult the (bit-)rate control.
