1 Description

This milestone introduces, PNG, the lossless image compression format used in this course for representing the videos. Basically, a (raster) digital video is a sequence of (digital) images (or frames). In this context, a frame can be described as a matrix of pixels (PIcture X-ray ELementS), arranged in a regular rectangular grid. Such sequences are usully captured by a CCD (Charge Coupled Device).

From a mathematical perspective, a frame sequence can be denoted by

{Vt,y,x ∈ ℙk,(t,y,x) ∈ ℕ3}

where t is a temporal coordinate, (y,x) is a spatial coordinate, and ℙk represents the (pixels) values domain, with dimension (number of pixel components) k [1].

Depending on k, we can have grayscale pixels (k = 1), color pixels (k = 3), and multispectral pixels (k > 3).1 Finally, if we take into consideration the number of bits/component that usually is 8 (although this value can be larger specially for grayscale frames), the pixel-depth will be 8 bits/pixel for grayscale sequences or 3 × 8 = 24 bits/pixel for color ones.

We are going to store a video as a sequence of ﬁles with names <preﬁx>000.png, <preﬁx>001.png, ..., where <preﬁx> speciﬁes the sequence name and “.png” indicates that the frames are coded using PNG (pronounced “ping”) image compression format [4, 2]. PNG uses DPCM and the DEFLATE entropy codec (Huﬀman coding + LZSS) [3] to remove the spatial and statistical redudancy. Notice that no color transform is used.

Finally, notice that all the frames of a sequence usually:

  1. Have the same spatial resolution.

  2. Are, at least originally, represented in the RGB domain.

  3. Depending on the frame, in PNG, each color component (channel) has a bit depth of 8 or 16 bits, representing an unsigned integer.
