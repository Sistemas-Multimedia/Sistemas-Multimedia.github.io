1 Description

This task introduces, PNG, the lossless image compression format used in this course for representing digital images and videos. Basically, a (raster) video is a sequence of images (or frames, in the context of video).

An image can be described as a matrix of pixels (PIcture X-ray ELementS), arranged in a regular rectangular grid. Such sequences are usully captured by a CCD (Charge Coupled Device). In maths, an image (or a frame) is \begin {equation} {\mathbf I} = \{{\mathbf I}_{y,x}\}\in \mathbb {P}^{k}, (y,x)\in \mathbb {N}^2\} \label {eq:image} \end {equation} where \((y,x)\) is a spatial coordinate, and \(\mathbb {P}^{k}\) represents the (pixels) values domain, with dimension (number of pixel components) \(k\) [1].

In a similar way, a sequence frames can be denoted by \begin {equation} {\mathbf V} = \{{\mathbf V}_{t,y,x}\in \mathbb {P}^{k}, (t,y,x)\in \mathbb {N}^3\} = \{{\mathbf I}_t\} \label {eq:video} \end {equation} where \(t\) is a temporal coordinate, and \(\mathbf I\) is an image such as the described in Eq. ??.

Depending on \(k\), we can have grayscale pixels (\(k=1\)), color pixels (\(k=3\)), and multispectral pixels (\(k>3\)).1 Finally, if we take into consideration the number of bits/component that usually is \(8\) (although this value can be larger specially for grayscale frames), the pixel-depth will be \(8\) bits/pixel for grayscale sequences or \(3\times 8=24\) bits/pixel for color ones.

In our labs, we are going to store a video as a sequence of ﬁles with names <prefix>000.png, <prefix>001.png, ..., where <prefix> speciﬁes the sequence name and “.png” indicates that the frames are coded using PNG (pronounced “ping”) image compression format [4, 2]. PNG uses DPCM and the DEFLATE entropy codec (Huﬀman coding + LZSS) [3] to remove the spatial and statistical redudancy. Notice that no color transform is used.

Finally, notice that all the frames of a sequence usually:

  1. Have the same spatial resolution.

  2. Are, at least originally, represented in the RGB domain.

  3. Depending on the frame, in PNG, each color component (channel) has a bit depth of \(8\) or \(16\) bits, representing an unsigned integer.
