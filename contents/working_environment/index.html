1 About the Operating System

1.1 Running/Developing VCF “Natively”

The VCF project [1] has been designed to work in Linux. Said that, it is also possible to run VCF in other operating systems such as OSX and Windows because Python [3] and Git [2], the two main tools that we will use, are also available. In the case of Linux, you can use your favorite distribution.

The developing of VCF in a native environment is the preferred option because is the most efficient one from a pure computational point of view.

1.2 Running/Developing CVF in a Linux Container

The second most efficient option in terms of computational requirements and speed is to run Linux in a Linux container, such as the provided by Docker (example for Ubuntu) or LXD. Notice however, that you should be able to run graphical applications (such as ImageMagick and Firefox) in your container.

Remember that in Windows there exists the posibility of using the Windows Subsystem for Linux (WSL).

1.3 Running/Developing CVF in a Linux Virtual Machine

VCF works also in any Linux installed in a virtual machine (VM) environment such as Virtual Box, VMware and Parallels. The only drawback of this way of running CVF is that you will require some extra memory and CPU power, compared to the containers.

Obviously, any remote host (virtualized or not) running Linux is completely adequate.

1.4 Running CVF in a “Notebook”

Finally, it is possible to run CVF services that able to run “notebooks”, such as Google Colaboratory or Binder. Notice, however, that in this case, you will probably need to configure your notebooks for installing all the necessary stuff from the scratch if you use these cloud services. On the contrary, if you opt for using Jupyter, this last action is not necessary.

2 About the Programming Language(s)

VCF is written in Python. You will need also to interact with a commands line terminal.

In any case, it is very convenient tu use a virtual environment.
