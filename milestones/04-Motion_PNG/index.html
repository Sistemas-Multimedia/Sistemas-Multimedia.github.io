1 Description

This milestone shows the structure of the videos used in the MRVC project. Basically, a (raster) digital video is a sequence of (digital) images (or frames). In the context of MRVC, an image can be described as a matrix of pixels (PIcture X-ray ELementS), arranged in a regular rectangular grid. Such sequences are usully captured by a CCD (Charge Coupled Device). In the digital world, a video can be denoted by

Vt,y,x

where the spatial coordinatinates (y,x) ∈ ℕ × ℕ, the temporal coordinate t ∈ ℕ, and Vt,y,x ∈P is the pixel value, where ℙk is the (pixels) values domain [2]. t indexes the t-th image in the video and its resolution is measured in FPS (Frames Per Second). (x,y) identify a pixel in an image and 0 ≤ x < the image width: 0 ≤ y < the image height.1

Depending on the k dimension of ℙ (the number of components of a pixel), we usually can have grayscale pixels (k = 1) or color (panchromatic) pixels (k = 3).2 Finally, if we take into consideration the number of bits/component (usually 8, although this value can be larger specially for grayscale images), the pixel-depth will be 8 bits/pixel for grayscale sequences or 3 × 8 = 24 bits/pixel for color ones.

We are going to store a video as a sequence of ﬁles with names <preﬁx>000.png, <preﬁx>001.png, ..., where <preﬁx> speciﬁes the video name and “.png” indicates that the frames are coded using the lossless PNG (pronounced “ping”) image format [4, 1]. PNG uses DPCM and the DEFLATE codec (Huﬀman coding + LZSS) [3] to remove the spatial and statistical redudancy. Notice that no color transform is used.

We will use PNG basically as an entropy codec, which is a combination of Huﬀman Coding and Lempel-Ziv-77 Coding. However, notice that any other lossless image compressing ﬁle format (such as TIFF) could be used.

Finally, notice that all the frames of the videos used in this subject:

  1. Have the same spatial resolution.
  2. Are represented in the RGB domain.
  3. Depending on the frame, each color component (channel) has bit depth of 8 or 16 bits, representing an unsigned integer.
