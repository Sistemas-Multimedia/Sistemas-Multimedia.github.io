1 Description

1.1 Components correlation

Usually, some part of the data in an image is redundant (removed without loss of information). In the case of a color image in the RGB domain, the three components of each pixel (each one measuring the energy in a diﬀerent band of the visible spectrum) can be correlated.

To estimate the redundancy we have basically two options:

  1. To compute the 0-order (memoryless source) entropy of the signal: the higher the entropy, the lower the redudancy. In fact, if we suppose that the samples of the signal are uncorrelated, the 0-order entropy is an exact measure of the expected bit-rate achieved by an arithmetic encoder (the most eﬃcient entropy compressor). Unfortunately, the 0-order entropy is usually only a lower bound of the redundancy estimation, i.e., lower bit-rates can be achieved in practice.
  2. A better way is to use an lossless compressor: the higher the length of the compressed ﬁle compared to the length of the original ﬁle, the lower the redundancy.1 Notice, however, that although this estimation is more accurate than the 0-order entropy, in general, it depends on the encoding algorithm (diﬀerent algoritms can provide diﬀerent estimations).

1.2 Color spaces

In this milestone, the intercomponent redundancy that may exist between the RGB components of each pixel of an image is removed. The color spaces that we will compare are:

  1. the RGB (the original one),
  2. the DCT (Discrete Cosine Transform) applied to the color domain, i.e., the color-DCT space,
  3. the YCrCb space, and
  4. the YCoCg space.

The last two spaces, which are luma-based (luminance-based2 color domains, rely on the idea of separating the luminance coeﬃcient s (Y) from two chroma coeﬃcients (red and blue in the case of YCrCb, orange and green in the case of YCoCg) and, instead of directly encoding the components, encoding the coeﬃcients, resulting from components diﬀerences. This transformation between color spaces, where the RGB components have been translated to three diﬀerent “color” subbands (of coeﬃcients) has two main advantages [1]:

  1. Compatibility with legacy black and white systems is maintained while at the same time the bandwidth of the signal can be optimized by using diﬀerent transmission bandwidths for the brightness and the chroma components.3
  2. Since the HVS (Human Visual System) is not able to perceive detail in the chrominance as well as it does in the luminance, the amount of information, and consequently bandwidth, used in the chrominance can be reduced to approximately 1/4 of the used for the luminance without a noticeable distortion (see Fig. 1). This fact is employed when compressing digital still images and is one of the reason why, for example, the JPEG codec converts RGB images to YCrCb. See also this notebook.

1.3 Transform coding

Generally speaking, quantization is more eﬀective (for the same bit-rate, the lossy compression ratios are higher) when the energy of the signal is accumulated in an small number of coeﬃcients because we can dedicate more bits to encode the more energetic ones.4 Transform coding can exploit correlation in signals to concentrate the energy in a subset of samples.

All linear5 transforms can be described as a matrix-vector product [4]

y = Kx, (1)

where x is the input signal, K is the analysis transform matrix, and y is the output decomposition. Notice that

yi = ⟨Ki,xi⟩, (2)

where Ki is the i-th row of K, and ⟨⋅,⋅⟩ denotes the inner product. This basically means that yi is proportional to the similarity between the input signal x and the taps of the ﬁlter Ki.6 The inverse (synthesis) transform is computed by

x = K−1y, (3)

where K−1 denotes to the inverse matrix of K. When K is orthonormal, it holds that

K = K−1 = KT, (4)

where KT represents the transpose matrix of K. Without considering scale factors, Eq. 4 is true also for all orthogonal transforms. Orthogonal and orthonormal transforms satisfy that

⟨Ki,Kj⟩ = 0,∀ ⁡i≠j. (5)

Transforms are used in signal coding to provide relative (between subbands) energy compaction. The capatility of a transform to achieve this can be estimated by the so called Transform Coding Gain [5, 3] deﬁned by

G = 1 N ∑ n=1Nσn2 (∏ n=1Nσn2) 1 N , (6)

where N is the number of coeﬃcients in a block (in our case, the number of coeﬃcients in a transformed pixel, i.e., N = 3), and σn2 is the variance of the n-th coeﬃcient in the block. As it can be seen, G is the ratio of the arithmetic mean of the variances of the transform coeﬃcients to their geometric mean. Notice that G is computed inside of a block (a pixel in the case of a color transform), not among blocks (pixels).

1.5 The RGB ⇔YCrCb transform

To convert a (color) pixel from the RGB domain into the YCrCb color domain, we use the RGB/YCrCb (analysis) transform [2]

Y = 0.299R + 0.587G + 0.114B Cr = 0.713(R −Y) + δ Cb = 0.564(B −Y) + δ, (12)

where,

δ = { 128 for 8 bits (unsigned) images, 32768 for 16 bits (unsigned) images, 0.5 for ﬂoating point ([0,1]) images (13)

is used to avoid negative coeﬃcients. As it can be seen, Cr and Cb are scaled versions of R −Y and B −Y, so Cr and Cb can be interpreted as measures of how much red and blue content in a pixel diﬀers from luma, respectively. Notice also that for a gray pixel, R = G = B = Y, and so Cr = Cb = 0 [2]. where Y is the luma and CrCb are the chromas. The main reason for such mapping is that the HVS is much less sensitive to the high-frequency information in the chromas [1]. Thus, as we mentioned above, compression systems such as JPEG can downsample the chromas (usually by 2:1 in the horizontal and vertical directions), as well as increase their quantization step sizes with respect to the luma, to achieve further compression without noticeable visual distortion [2].

As it can be seen, considering that the RGB values ranges between 0 and 255 (and rounding to the nearest integer), 0 ≤Y ≤ 255, 0 ≤Cr ≤ 255 and 0 ≤Cb ≤ 255, and therefore, the number of bits that are necessary to represent each YCrCb component is 8 (although we must use ﬂoating point arithmetic to perform the transform).

Finally, notice that the YCrCb transform is not orthogonal because the analysis ﬁlters are not independent. This can be seen in the Eq. 12, where the Cr coeﬃcients depend on the coeﬃcients of Y, and therefore, there is a dependency between both basis, and something similar happens for the Cb subband. This can be also easely checked:

0.299 ∗ 0.5 + 0.587 ∗ (−0.4187) + 0.114 ∗ (−0.0813) = −0.1055451≠0,

0.299 ∗ (−0.1687) + 0.587 ∗ (−0.3313) + 0.114 ∗ 0.5 = −0.1879144≠0, and

0.5 ∗ (−0.1687) + (−0.4187) ∗ (−0.3313) + (−0.0813) ∗ 0.5 = 0.01371531≠0.

The inverse (synthesis) transform is deﬁned by

R = Y + 1.403(Cr − δ) G = Y − 0.714(Cr − δ) − 0.344(Cb − δ) B = Y + 1.773(Cb − δ). (14)

that in matrix form is

[ R G B ] = [ 1 1.403 0 1 −0.714 −0.344 1 0 1.773 ] [ Y Cr − δ Cb − δ ], (15)

1.5.1 Quantization in the YCrCb domain

The YCrCb transform is not orthogonal and the relative synthesis ﬁlters gains depends on the energy of the inversely transformed components. In this case, we can estimate the distortion generated by the quantization of a color subband, always measured in the RGB domain, if the rest of subbands are unquantized. This can be the algorithm:

  1. Variying the Δ, estimate the RD curve for each YCrCb subband, keeping the other subbands unquantized. The distortion must be measures in the RGB domain.
  2. Sort the RD points by their slope.
  3. Apply progressively the combinations quantization steps. The distortion can be measure in both, the color-DCT and the RGB domains.

Notice that the only alternative to this “fast” rate-control algorithm is to perform a brute-force search of quantization steps combinations.

1.6 The RGB ⇔YCoCg transform

Clearly, orthogonality is a desired property in lossy compression systems because it helps to isolate8 the impact (in the case of a color transform) of each color subband on the quality of the reconstruction of the image, simplifying signiﬁcantly the determination of the pattern of QSs that generate the optimal RD curve.

Moreover, Eqs. ?? and 14 were derived by Principal Component Analysis (PCA) on old9 video data. The same procedure has been carried out with newer10 images, obtaining

[ Y C 1 C2 ] = [ 1 3 1 3 1 3 1 2 0 −1 2 −1 4 1 2 −1 4 ] [ R G B ] ⇔ [ R G B ] = [ 1 1 −2 3 1 0 4 3 1 −1 −2 3 ] [ Y C 1 C2 ] . (16)

The YCoCg color transform is orthogonal and the synthesis ﬁlters gains11 are 22/9 (for Y), 25/9 (for C1) and 22/9 (for C2).

Unfortunately, from a perceptual perspective we must impose (thinking of the subsampling of the chromas) some features in a color transform (such as the inﬂuence of the green channel on the luma channel should be high) that violates the orthogonality constrain [2]. For this reason the authors ﬁnally propose the transform

[ Y Co Cg ] = [ 1 4 1 2 1 4 1 2 0 −1 2 −1 4 1 2 −1 4 ] [ R G B ] ⇔ [ R G B ] = [ 1 1 −1 1 0 1 1 −1 −1 ] [ Y Co Cg ] , (17)

that is near orthogonal12 , and has channel gains 3 (Y), 2 (Co) and 3 (Cg).

Again, notice that if the RGB values ranges between 0 and 255 (and rounding to the nearest integer), 0 ≤Y ≤ 255, − 128 ≤Co ≤ 127 and − 128 ≤Cg ≤ 127, and therefore, the number of bits that are necessary to represent each component is 8. Therefore, we can use the same QS range for each component. See this notebook.
